# 🕵️‍♂️ 謎解きアプリ (Mystery App)

Spring Boot の学習と復職リハビリを兼ねて開発中の Web ブラウザで遊べる謎解きゲームです。
アーケードゲームのように、サクッと遊んでスコアを競えるシステムを目指しています。

## 🛠 技術スタック

- **Language:** Java 21
- **Framework:** Spring Boot 3
- **Template Engine:** Thymeleaf
- **Infrastructure:** Docker / Render
- **Design:** Mermaid (画面遷移図など)
- **Tools:** Jira

## 📏 開発ルール (Commit Guidelines)

### 🌿 ブランチ

原則として `main` ブランチには直接コミットせず、作業用ブランチを作成して開発します。

**ブランチ名の命名規則:** `prefix/Jiraの課題キー-内容`
（Prefix はコミットルールに準拠します）

- `feat/KAN-XXX-top-page` (機能追加)
- `fix/KAN-XXX-login-error` (バグ修正)
- `docs/KAN-XXX-update-readme` (ドキュメント更新)

### コミット

コミットメッセージは **Conventional Commits** に基づき、以下のプレフィックスを付けて統一します。  
また、**Jira の課題キー** を含めることでチケットと自動連携させます。

**フォーマット:** `prefix: 課題キー コミット内容`

| Prefix       | 意味            | 使いどき                                                   |
| :----------- | :-------------- | :--------------------------------------------------------- |
| **feat**     | ✨ 新機能       | 新機能追加や仕様変更があった時                             |
| **fix**      | 🐛 バグ修正     | 動かないコードやバグを直した時                             |
| **docs**     | 📚 ドキュメント | README や画面遷移図(design フォルダ)を更新した時           |
| **style**    | 🎨 見た目       | CSS 修正やコードのフォーマット整形（動作に影響しないもの） |
| **refactor** | ♻️ リファクタ   | 機能を変えずにコードを綺麗にした時                         |
| **test**     | 🧪 テスト       | JUnit などのテストコードを追加・修正した時                 |
| **chore**    | 🔧 雑用         | Dockerfile の設定やライブラリ更新など                      |

#### コミット例

- `feat: KAN-XXX 第1問の画面を作成`
- `docs: KAN-XXX 画面遷移図を更新`
- `fix: KAN-XXX 誤字を修正`  
  細かい変更など記述したいことがあれば一つ空行を入れて記載するようお願いします。
  docs の場合チケットに紐づいていない場合があるので、課題キーがなければ記載しないで大丈夫です。

## 📂 ディレクトリ構成

- `src/`: ソースコード本体
- `docs/`: 設計図やタスク管理ドキュメント
  - `design/`: 画面遷移図など
  - `todo/`: タスクリスト
